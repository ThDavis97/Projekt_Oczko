config\+USE\+\_\+\+TASK\+\_\+\+NOTIFICATIONS must be undefined or defined as 1 for this macro to be available.

When config\+USE\+\_\+\+TASK\+\_\+\+NOTIFICATIONS is set to one each task has its own private \char`\"{}notification value\char`\"{}, which is a 32-\/bit unsigned integer (uint32\+\_\+t).

Events can be sent to a task using an intermediary object. Examples of such objects are queues, semaphores, mutexes and event groups. Task notifications are a method of sending an event directly to a task without the need for such an intermediary object.

A notification sent to a task can optionally perform an action, such as update, overwrite or increment the task\textquotesingle{}s notification value. In that way task notifications can be used to send data to a task, or be used as light weight and fast binary or counting semaphores.

\doxyref{x\+Task\+Notify\+Give()}{p.}{task_8h_ac60cbd05577a3e4f3c3587dd9b213930} is a helper macro intended for use when task notifications are used as light weight and faster binary or counting semaphore equivalents. Actual Free\+RTOS semaphores are given using the \doxyref{x\+Semaphore\+Give()}{p.}{semphr_8h_aae55761cabfa9bf85c8f4430f78c0953} API function, the equivalent action that instead uses a task notification is \doxyref{x\+Task\+Notify\+Give()}{p.}{task_8h_ac60cbd05577a3e4f3c3587dd9b213930}.

When task notifications are being used as a binary or counting semaphore equivalent then the task being notified should wait for the notification using the ul\+Task\+Notification\+Take() API function rather than the \doxyref{x\+Task\+Notify\+Wait()}{p.}{task_8h_a0475fcda9718f403521c270a7270ff93} API function.

See {\texttt{ http\+://www.\+Free\+RTOS.\+org/\+RTOS-\/task-\/notifications.\+html}} for more details.
